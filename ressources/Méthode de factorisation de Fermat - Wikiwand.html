Mthode de factorisation de Fermat

Eric W. Weisstein Mthode de factorisation d'Euler Congruence de carrs

Un article de Wikipdia, l'encyclopdie libre.

En arithmtique modulaire, la mthode de factorisation de Fermat est un algorithme de dcomposition en produit de facteurs premiers d'un entier naturel.

Intuition

L'intuition est la suivante. Tout entier naturel impair N se dcompose en la diffrence de deux carrs : N = a2 b2. Algbriquement, cette diffrence se factorise en (a + b)(a b) et, si ni a + b ni a b n'est gal 1, alors ce sont des facteurs non triviaux de N.

Il existe une telle reprsentation pour tout nombre impair compos. Si N = cd est une factorisation de N, alors N = ( c + d 2 ) 2 ( c d 2 ) 2 . {\displaystyle N=\left({\frac {c+d}{2))\right)^{2}-\left({\frac {c-d}{2))\right)^{2}.} Puisque N est impair, c et d le sont aussi et ces  moitis  sont des nombres entiers. Notons qu'un multiple de 4 donne aussi une diffrence de deux carrs, en posant c et d comme nombres pairs.

Dans sa forme la plus simple, la mthode de factorisation de Fermat peut tre plus lente que la factorisation par essais de divisions. Nanmoins, la combinaisons des deux mthodes est plus efficace qu'uniquement l'une ou l'autre.

Mthode de base

On essaie diffrentes valeurs de a, en esprant que a2 N soit un carr b2. On peut utiliser qu'un carr ne peut se terminer que par 0, 1, 4, 5, 6 ou 9 dans le systme dcimal.

FactorFermat(N): // N doit tre impair
A arrondi(sqrt(N))
Bsq A*A N
tant que Bsq n'est pas un carr:
A A + 1
Bsq A*A N // ou de faon quivalente : Bsq Bsq + 2*A + 1
fin tant que
retourner A sqrt(Bsq) // ou A + sqrt(Bsq)

Analyse

Soit N un entier possdant plus d'une factorisation (N possde plus de deux facteurs). La mthode de Fermat donne la factorisation de N avec les plus petites valeurs de a et de b, c'est--dire que a + b est le plus petit facteur suprieur la racine carre de N. Donc, a b = N/(a + b) est le plus grand facteur n'excdant pas la racine carre de N. Si la mthode produit N = 1 N, cela signifie que N est un nombre premier.

Complexit

Pour N = cd, soit c le plus grand facteur plus petit que la racine carre de N et soit a = (c + d)/2. Alors, le nombre d'oprations est approximativement c + d 2 N = ( d c ) 2 2 = ( N c ) 2 2 c . {\displaystyle {\frac {c+d}{2))-{\sqrt {N))={\frac {({\sqrt {d))-{\sqrt {c)))^{2)){2))={\frac {({\sqrt {N))-c)^{2)){2c)).}

Si N est premier (donc c = 1), l'algorithme prend O(N) oprations. C'est donc une faon trs inefficace de dmontrer la primalit d'un nombre. Par contre, si N a un facteur suffisamment prs de sa racine carre, alors la mthode de Fermat est trs efficace. Plus prcisment, si c diffre de moins que (4N)1/4 de N, la mthode ne prend qu'une seule opration. Cette analyse est indpendante de la grandeur de N.

Exemple

Par exemple, pour factoriser N = 5959 (N 77,19), on calcule :

A 78 79 80
Bsq 125 282 441

Le troisime essai produit un carr : A = 80, B = 21 et les facteurs sont A B = 59 et A + B = 101.

Amliorations importantes

Les mthodes de factorisation du crible quadratique et du crible gnral de corps de nombres (GNFS) sont bases en grande partie sur la mthode de factorisation de Fermat.

Mthode de factorisation de Lehman

Article dtaill : Mthode de factorisation de Lehman (de).

La mthode de Fermat fonctionne optimalement lorsqu'un des facteurs est approximativement la racine carre de N. La question qui s'ensuit est : peut-on s'arranger pour que ce soit le cas ?

Si on connat approximativement le quotient de deux facteurs, d/c, alors on peut choisir un nombre rationnel, v/u, proche de ce quotient. Posons Nuv = cv du, donc les facteurs sont peu prs gaux : la mthode de Fermat applique Nuv les trouvera rapidement. Puis, on obtient pgcd(N, cv) = c et pgcd(N, du) = d (sauf si c divise u ou d divise v).

En gnral, le quotient n'est pas connu, mais on peut essayer diffrentes valeurs de u/v et essayer de factoriser chaque Nuv rsultant. Une mthode systmatique est donne par Russell Sherman Lehman[1]. Sa complexit calculatoire est de O(N1/3+), laquelle est avantageuse par rapport celle pour la mthode des divisions successives, dont la complexit est en O(N1/2+).
